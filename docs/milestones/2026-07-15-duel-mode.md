# Milestone: Duel mode (v0.11.0)

**Shipped:** wild opponents that ambush you during real work, fight your pet
with its actual techniques, deplete an HP bar that finally means something,
and leave a mark — EXP, a ⚔ dex entry, or a fainted pet.

## What was built

- **Pre-computed battle scripts.** The moment an encounter triggers,
  `pet-core.sh` generates the *entire* fight into `~/.cache/claude-pokemon-pet/duel.json`:
  opponent, every turn's damage roll, both HP tracks, the result. The
  overlay, terminal renderer, and statusline replay that script by wall
  clock. No daemons, no timers, no renderer-side game logic — the
  architecture invariant ("core reduces, renderers view") survives combat.
- **Encounters.** ~1 per 4 completed tasks (seeded roll), capped at 3/day,
  plus `claude-pokemon-pet duel` / `/pet duel` on demand. Opponents are
  same-franchise and stage-matched: a rookie pet meets rookies (digimon
  walk the canonical evolution graph; pokémon index the line arrays).
- **Real HP.** The old `100 − 15·mistakes + 10·tasks` formula — a gauge
  that could never meaningfully drop — is gone. HP now lives in a
  date-stamped file: duels deplete it (damage persists after the fight!),
  each completed task heals +10, a lost duel faints the pet until your next
  task revives it at 60, and each day starts fresh at 100.
- **Stakes.** Win → +1 level (real EXP that feeds evolution gates) and the
  defeated wild mon lands in your dex with a ⚔ marker (cleared if you ever
  raise that species yourself). Lose → faint. All-time ⚔W–L record replaces
  the streak flame next to the pet's name, in the statusline, and on the
  trainer card.
- **Choreography, per the attack-FX hard rules.** Challenger slides in from
  the right; split HP bars; attacker lunges; element-tinted star glints
  burst at the *defender*; victory hop or fainting dim-out. Terminal shows
  the same fight as side-by-side half-block sprites (pane ≥ 60 cols) with a
  battle log and paired HP bars. **No projectiles anywhere.**

## How it works under the hood

1. `cmd_event done` → counters bump under `counter_lock` → heal/revive →
   resolve → encounter roll (also under `counter_lock` — see below).
2. Generation is one jq program (`DUEL_JQ`) fed by a tiny LCG
   (`(x·75+74) mod 65537` — constants stay small because jq numbers are
   IEEE doubles). Same seed → byte-identical fight; `PET_SEED` makes every
   test deterministic.
3. `duel.json` is **language-neutral**: opponent name and move are stored
   in both locales; `RESOLVE_JQ` flattens them to the active language when
   embedding the duel into `resolved.json`, so switching 한국어/English
   mid-fight re-localizes cleanly.
4. The first resolve after `end_ts` applies the outcome exactly once —
   `.duel-apply.lock` (mkdir) with an atomic `applied: true` rewrite before
   release, so concurrent hooks can skip-on-busy safely. A duel left over
   from yesterday is discarded unapplied.

## Review findings that shaped the code

The adversarial review (code-reviewer, run twice to PASS) caught two real
bugs by *reproducing* them:

- **Encounter generation raced.** 20 concurrent `done` hooks blew the 3/day
  cap to 6 because generation ran outside `counter_lock`. Now the whole
  check-roll-generate-bump sequence holds the lock; a 20-way regression
  test pins `duels_today == 1`.
- **jq's `//` operator treats `false` as absent.** `.applied // "default"`
  silently rewrote `applied: false`, which made the core delete every fresh
  duel as "corrupt". Boolean reads now use explicit comparisons; the lesson
  is recorded in CLAUDE.md.
- Plus: jq 1.6 (Debian) rejects variables that shadow keywords (`$try`,
  `$end`) — parses fine on jq 1.7/macOS, caught only by the container run.

## Verification

- **268 bash assertions** (61 of them in the new `tests/test-duel.sh`:
  generation determinism, cap, faint/revive, apply-exactly-once under
  20-way concurrency, rollover discard, localization switch, garbage-file
  self-heals) + **54 Python unit tests** (incl. duel phases, captions,
  josa 이/가, sprite joining, fainted).
- Full suite green on macOS (bash 3.2 + jq 1.7) **and** Debian bookworm
  (bash 5 + jq 1.6, GNU userland) in a container.
- Live QA: a scripted 23-second fight replayed on the real overlay
  (slide-in, lunges, sparks at the defender, victory beat, layout restore)
  and headless through the terminal renderer (frame counts per phase
  verified); state backed up and restored.
